The backslash (\) is an escape character, which is also commonly used in other programming languages. It basically tells the Python interpreter to render a special character with some special meaning. This is why using them as directory separators would result in errors because Python would mistaken those escape sequences as special characters.

Commonly used escape sequences are:

Escape Sequence Meaning
\t tab
\' single quote
\" double quote
\n newline

Lists¶

Lists are one of the most useful and versatile data types in Python. They are similar to arrays in Javascript. Operations on lists are similar to the operations done on strings, except that elements in a list are more like objects, rather than characters of a string. Lists are defined using square brackets [].

Tuples¶

Tuples are similar to Python lists. The only difference is that tuples are immutable, which means that once they are defined in memory, they cannot be altered. Therefore, methods like append, insert, remove, reverse and sort will not work. Tuples use parentheses () to be defined.

Dictionaries¶

Dictionaries are another commonly used data type in Python. They are mapping types, which means that every element in a dictionary is a key-value pair. This makes them ideal for doing data manipulations or used as intermediate data structures for database operations. They are very similar to JSON data structures found in Javascript applications and NoSQL databases such as MongoDB. Dictionaries have unique keys, where their values can be any object. If a key occurs more than once, the last value for that key becomes the corresponding value in the new dictionary. Dictionaries use curly brackets {} to be defined, and the key and value are separated by a colon :.

In [3]:
mapper = {'Name': 'Purrsephone',
          'Occupation': ['Greek Goddess', 'Cat Lover'],
          'Age': 9999}
print mapper

{'Age': 9999, 'Name': 'Purrsephone', 'Occupation': ['Greek Goddess', 'Cat Lover']}

Note: Notice how the keys aren't ordered in the way they were inserted?

Retrospective Usage of List¶

Earlier, I had created a nested list with the same kind of information that I had used in the dictionary. While it is possible to obtain the Name, Occupation, and Age values from the nested list, I would have to first determine the index positions of each of those "keys". In addition, the list doesn't enforce a unique rule on the "key" list ['Name', 'Occupation', 'Age'] so the dictionary would be a better choice if this information is to be inserted in a database later.

In [8]:
print nested_list  # here's the nested list
print mapper  # here's the dictionary

[['Name', 'Occupation', 'Age'], ['Purrsephone', ['Greek Goddess', 'Cat Lover'], 28]]
{'Age': 28, 'Name': 'Purrsephone', 'Occupation': ['Greek Goddess', 'Cat Lover']}

# To get the occupation
