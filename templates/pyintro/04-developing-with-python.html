IDE¶

While Jupyter is a nice tool for developing and testing Python code, you will eventually need to utilize an integrated development environment (IDE) for building more complicated apps. An IDE provides an editor, a debugger, version control, and real-time code quality checks under one development environment. The biggest advantage of using an IDE is its support for cross-technology development. Even if your job may solely be developing statistical models in Python, those models will eventually have to be deployed and integrated in an application/environment that utilizes other technologies.

There are many Python IDEs currently available. I'll just cover two:

    • Text editor with high customizability

Virtualenv¶

Motivation¶

In a collaborative environment, different projects may require different Python dependencies. In addition, a production environment may have different library requirements compared to a development environment. The solution to conflicting environment dependencies is virtualenv, which creates an isolated Python environment for development/test/deployment. Consider the following project (project X) with the following dependencies:

Flask==0.10.1
Flask-SQLAlchemy==2.0
Flask-WTF==0.12
Jinja2==2.8
MarkupSafe==0.23
SQLAlchemy==1.0.8

What happens if a different project (project Y) depends on Jinja2 v3.0? Virtualenv will solve the problem by creating an environment for that project. As a matter of fact, project X should have its own environment. This prevents conflicts between different project dependencies, as well as keeping the global environment clean.

Any other benefits?

Usage¶

  • First make sure that virtualenv is installed. With PowerShell open, type:
pip install virtualenv
  • Then cd into your project directory and create a virtual environment:
cd "C:/path/to/MyProject"
virtualenv venv
  • Then activate the virtual environment:
venv/Scripts/activate
  • You will know that the virtual environment has been activated when (venv) precedes the console prompt.
  • You will need to install whatever packages are required for your package into that virtual environment since it is isolated from the global environment.
  • To deactivate the virtual environment, type:
deactivate

Coding Conventions¶

When coding in Python, it is beneficial to follow certain coding conventions that make it easier to read code. In addition, following coding conventions make it easier for collaboration, as contributors to the project will already expect and be comfortable with the coding convention.

Python's coding conventions are detailed by standards called PEP (Python Enhancement Proposal). There are various PEPs that provides guides to how to write code. PEP 8 contains coding style guidelines, while PEP 257 contains documentation guidelines.

Modern Python IDEs like PyCharm can enforce these PEP conventions.

Unit Testing and TDD¶

When developing large projects, one important aspect that is usually overlooked is testing. Testing gives confidence to the developer and in the long term, is an essential benefit to the project lifetime.

One method of testing that is commonly used for development is unit testing, which involves testing the smallest parts of a program (e.g. function). The test involves writing multiple test cases, as well as edge cases, against that part to ensure that it behaves properly. Unit tests are usually automated, but can be done manually. Python includes a unittest library for writing unit tests.

Unit testing is a component of Test Driven Development (TDD), which is a development process that uses test cases for developing the program. TDD works well in Agile environments, since requirements and feature enhancements are broken down into test cases, which then drive the development process of project.
